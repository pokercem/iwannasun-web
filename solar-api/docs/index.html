Developer Guide

Integrate /v1/features in a controlled beta workflow.

/v1/features returns production-derived solar feature signals as a timestamped timeline. It is designed as a feature layer for downstream forecasting, PV modeling, and operational analytics systems.

Endpoint Overview

Method: GET

Endpoint: https://api.iwannasun.com/v1/features

Access model: approved beta integrations using request token header.

Output: meta, feature_schema, and timestamped timeline rows.

Authentication

For approved beta access, send the request token in header X-IWS-Feature-Token.

X-IWS-Feature-Token: YOUR_TOKEN

Full curl example

curl \
  -H "X-IWS-Feature-Token: YOUR_TOKEN" \
  "https://api.iwannasun.com/v1/features?lat=39.74&lon=-104.99&days=1&model=local&mode=full"

Request Parameters

  • lat: latitude in decimal degrees (-90..90), required.
  • lon: longitude in decimal degrees (-180..180), required.
  • days: forecast horizon in days (1..3), default 1.
  • model: geometry mode (local|ray), default local.
  • mode: timeline density (full|quick), default full.
  • source: optional, currently only production.

Errors and Limits

Error responses include a structured error object and a request trace id.

{
  "detail": "Feature access token required.",
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Feature access token required.",
    "request_id": "..."
  }
}

Common error codes

  • INVALID_ARGUMENT: invalid query parameter values.
  • UNAUTHORIZED: access token missing when token gate is enabled.
  • FORBIDDEN: access token present but invalid.
  • API_RATE_LIMIT: request rate exceeded.
  • UPSTREAM_RATE_LIMIT: upstream forecast provider is rate-limiting requests.
  • UPSTREAM_BAD_RESPONSE: upstream response could not be used.
  • TEMPORARY_UNAVAILABLE: upstream/provider issue.
  • INTERNAL_ERROR: unhandled server error.

Rate limit behavior

Requests are rate-limited per client IP. Default server configuration is 120 requests per 60 seconds (configurable).

When limited, the API returns 429 with Retry-After and X-IWS-Error-Code: API_RATE_LIMIT.

Integration Checklist

  1. Request beta access token and store it as a secret.
  2. Integrate /v1/features with token header and retry handling for 429/503.
  3. Start with core fields and validate downstream lift before adding advanced fields.
  4. Log request_id for traceability and support diagnostics.

Validation Summary

  • SURFRAD irradiance benchmark: ~11–20% DNI RMSE reduction in broken-cloud and cloud/DNI disagreement regimes using NOAA SURFRAD ground irradiance stations.
  • PVDAQ plant-power backtest: ~2% overall RMSE improvement across 8 NREL PVDAQ solar plants (~6 months of historical data).
  • Station-holdout and time-holdout validation protocols were used.
  • Analysis focused on broken-cloud and transition regimes where beam-state uncertainty is highest.
  • Current product framing is a feature layer for downstream models, not a forecast replacement.
