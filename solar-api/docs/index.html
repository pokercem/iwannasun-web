Developer Guide

Integrate /v1/features with clear auth, schema, and guardrail behavior.

Benchmark Context

  • SURFRAD 2025 ray-mode validation.
  • 7 stations, 30,500 daylight rows.
  • Main run plus time holdout, station holdout, and signal analysis.

Endpoint Overview

Method: GET

Endpoint: https://api.iwannasun.com/v1/features

Output: meta, feature_schema, and timestamped timeline rows.

Authentication

When token gate is enabled server-side, use either method:

  • Header: X-IWS-Feature-Token: YOUR_TOKEN
  • Query parameter: apikey=YOUR_TOKEN

Header auth example

curl \
  -H "X-IWS-Feature-Token: YOUR_TOKEN" \
  "https://api.iwannasun.com/v1/features?lat=39.74&lon=-104.99&days=1&model=ray&mode=full"

Query auth example

https://api.iwannasun.com/v1/features?lat=39.74&lon=-104.99&days=1&model=ray&mode=full&apikey=YOUR_TOKEN

Request Parameters

  • lat: latitude in decimal degrees (-90..90), required.
  • lon: longitude in decimal degrees (-180..180), required.
  • days: forecast horizon in days (1..3), default 1.
  • model: geometry input mode (local|ray) for the same scorer, default local.
  • mode: timeline density (full|quick), default full.
  • source: optional, currently only production.

Mode guidance: use model=ray by default for quality-sensitive integrations; use model=local when you need a simpler fallback path.

Request example

curl "https://api.iwannasun.com/v1/features?lat=39.74&lon=-104.99&days=1&model=ray&mode=full"

Field Explanations

Core fields

  • sun_score: final score (0..100).
  • score_cloud: cloud component score (0..100).
  • score_dni: DNI component score (0..100).
  • disagree: cloud-vs-DNI disagreement (0..1).
  • broken_cloud_flag: broken-cloud regime marker.
  • high_disagreement_flag: true when disagree >= 0.35.

Advanced fields

  • confidence, w_dni, dni_ratio, cloud_transmission.
  • instability, low_sun_flag, horizon_flag, fragility_proxy.
  • dni_limiter_activated, low_cloud_cap_activated, horizon_rescue_activated, late_caps_activated.

Guardrail Guidance

Use high_disagreement_flag as a lower-reliability indicator.

  • Definition: high_disagreement_flag = (disagree >= 0.35).
  • When true, lower trust and apply fallback or dual-report handling.

Errors and Limits

Error responses include a structured error object and a request trace id.

{
  "detail": "Feature access token required.",
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Feature access token required.",
    "request_id": "..."
  }
}

Common error codes

  • INVALID_ARGUMENT
  • UNAUTHORIZED
  • FORBIDDEN
  • API_RATE_LIMIT
  • UPSTREAM_RATE_LIMIT
  • UPSTREAM_BAD_RESPONSE
  • TEMPORARY_UNAVAILABLE
  • INTERNAL_ERROR

Default API rate limit is 120 requests per 60 seconds (configurable).

Integration Notes

  1. Use header auth for server-to-server; use apikey for browser/no-code constraints.
  2. Handle 429 and 503 with retries/backoff.
  3. Start with core fields, then add advanced fields based on measured value.
  4. Log request_id for traceability.
