Solar Feature API

Beam-aware solar signals for improving solar forecast reliability.

Call /v1/features to retrieve production-derived beam-state, cloud-geometry, and DNI-disagreement signals. Each timestamped row exposes interpretable features that allow downstream systems to apply residual correction, fragility detection, and regime-aware forecast logic.

Signals are evaluated against SURFRAD ground measurements to validate beam-blocked interval detection under real irradiance conditions.

curl "https://api.iwannasun.com/v1/features?lat=39.74&lon=-104.99&days=1&model=local&mode=full"

Beam Visibility as a Forecasting Constraint

Direct beam visibility is a key driver of DNI and downstream PV output. In broken-cloud and low-sun regimes, beam availability can switch quickly and destabilize short-horizon forecasts.

Cloud-geometry path cues and local DNI-ratio cues often diverge in these intervals. That divergence is a practical fragility signal for identifying when baseline forecasts are least reliable and targeted correction is needed.

Use Cases

  • Solar forecasting models: improve residual correction during broken-cloud beam transitions.
  • PV production prediction: reduce forecast error during rapid cloud-driven irradiance changes.
  • Grid operations: identify fragile beam conditions ahead of short-horizon irradiance swings.
  • Solar analytics: evaluate model behavior by beam state and cloud regime.

Integration

Method: GET Endpoint: /v1/features

Access: optional token gate (header required only when enabled server-side).

The API exposes interpretable intermediate signals rather than a black-box forecast.

Request parameters

  • lat (required, -90..90)
  • lon (required, -180..180)
  • days (optional, 1..3 forecast days, default 1)
  • model (optional, local|ray, default local)
  • mode (optional, full|quick, default full)

Response cadence

  • Timeline interval: 10-minute timesteps.
  • mode=full: complete local-day timeline for each requested day.
  • mode=quick: short near-term timeline for low-latency clients.

What /v1/features Provides

A structured response with request meta, feature_schema, and per-timestamp rows in timeline that carry core and advanced signals.

Schema Stability (v1)

The /v1/features schema is stable within v1. New fields may be added, but existing fields will not change semantics without a version update.

Core signals (API v1)

sun_score
Final production sun visibility score (0-100).
score_cloud
Cloud-geometry component score (0-100).
score_dni
DNI-ratio component score (0-100).
disagree
Normalized cloud-vs-DNI disagreement (0-1).
sun_score_minus_score_dni
Final score delta vs DNI component.
score_cloud_minus_score_dni
Cloud-vs-DNI component delta.
sun_score_minus_score_cloud
Final score delta vs cloud component.
broken_cloud_flag
Broken-cloud regime indicator.

Trust and Validation

API v1 signals are computed from production model logic and evaluated in SURFRAD-style experiments. Validation focused on beam/occlusion behavior, residual add-value, and transfer across station and time holdouts.

The endpoint exposes intermediate production signals rather than a black-box forecast output. Downstream systems can combine these signals with existing models to apply targeted residual correction and regime-aware decision logic.

Validation Snapshot

  • Evaluated against SURFRAD ground irradiance measurements using production model signal outputs.
  • Tested with station-holdout and time-holdout validation protocols.
  • Beam-state and disagreement signals consistently identify forecast-fragile intervals.
  • Residual-correction usefulness is strongest during broken-cloud and transition regimes.

Validation Snapshot

  • SURFRAD irradiance benchmark: ~11–20% DNI RMSE reduction in broken-cloud and cloud/DNI disagreement regimes using NOAA SURFRAD ground irradiance stations.
  • PVDAQ plant-power backtest: ~2% overall RMSE improvement across 8 NREL PVDAQ solar plants (~6 months of historical data).

Signals appear strongest at the irradiance correction layer; plant-level gains depend on downstream forecasting and plant characteristics.
