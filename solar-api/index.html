Solar Feature API

Decision-ready solar signals for short-horizon forecasting and trading.

Raw weather feeds describe atmospheric conditions, but they do not tell forecasting systems when irradiance predictions are fragile.

The /v1/features API adds beam-state evidence and disagreement signals so forecasting systems can adjust confidence, residuals, and operational risk in real time.

Built for solar forecast engineers, intraday power traders, and energy analytics teams integrating PV and DNI forecasting workflows.

What It Provides

  • Beam-state and cloud/DNI evidence signals at 10-minute resolution.
  • Not raw weather fields. Derived, model-ready signals (sun_score, score_cloud, score_dni, disagree, high_disagreement_flag) designed for operational forecasting decisions.
  • high_disagreement_flag marks lower-reliability intervals so teams can gate trading decisions and flag fragile forecast windows.
  • Stable schema that plugs into residual correction and short-horizon forecast confidence controls.

Benchmark Evidence

  • Dataset: SURFRAD core stations.
  • Coverage: 7 stations, full-year 2025, 30,500 daylight rows.
  • Protocol: main benchmark, time holdout, station holdout, and signal analysis (ray mode).

Strengths observed

  • Stronger DNI regression and residual-signal add-value.
  • Improved breakthrough timing behavior.
  • Higher ROC/AUC discrimination in beam-state classification tasks.

Tradeoffs observed

  • Thresholded classification F1 is weaker in some regimes.
  • High-disagreement occlusion intervals remain difficult.

Typical Uses

  • Residual correction: adjust baseline DNI/PV forecasts when cloud and DNI evidence diverge.
  • Reliability gating: down-weight or flag intervals when high_disagreement_flag=true.
  • Ramp operations: trigger monitoring or alerts around likely cloud breakthrough events.
  • Model features: feed interpretable signals into short-horizon forecasting models.
