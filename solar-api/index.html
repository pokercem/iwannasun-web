Solar Feature API

Beam-aware solar signals for forecast correction.

Call /v1/features to retrieve production-derived beam-state, cloud-geometry, and DNI-disagreement signals. Each timestamped row exposes interpretable features that allow downstream systems to apply residual correction, fragility detection, and regime-aware forecast logic. 

curl "https://api.iwannasun.com/v1/features?lat=39.74&lon=-104.99&days=1&model=local&mode=full"

Use Cases

  • Solar forecasting models: improve residual correction in beam-transition regimes.
  • PV production prediction: reduce error during rapid cloud-driven irradiance changes.
  • Grid operations: surface fragile intervals ahead of short-horizon variability.
  • Solar analytics: evaluate model behavior by beam state, cloud regime, and solar elevation.

Beam Visibility as a Forecasting Constraint

Direct-beam visibility is a primary control on DNI and downstream PV output, but it changes abruptly under broken cloud and at low solar elevation. Small beam-state errors can amplify into larger operational forecast error.

Cloud-geometry cues and local DNI-ratio cues often diverge in these regimes. That divergence is a measurable stability signal: it identifies intervals where baseline forecasts are least reliable and targeted correction is most valuable.

Integration

Method: GET Endpoint: /v1/features

Access: currently provided for approved beta integrations (token-based).

Request parameters

  • lat (required, -90..90)
  • lon (required, -180..180)
  • days (optional, 1..3 forecast days, default 1)
  • model (optional, local|ray, default local)
  • mode (optional, full|quick, default full)

Response cadence

  • Timeline interval: 10-minute timesteps.
  • mode=full: complete local-day timeline for each requested day.
  • mode=quick: short near-term timeline for low-latency clients.

What /v1/features Provides

A structured response with request meta, feature_schema, and per-timestamp rows in timeline that carry core and advanced signals.

Core signals (API v1)

sun_score
Final production sun visibility score (0-100).
score_cloud
Cloud-geometry component score (0-100).
score_dni
DNI-ratio component score (0-100).
disagree
Normalized cloud-vs-DNI disagreement (0-1).
sun_score_minus_score_dni
Final score delta vs DNI component.
score_cloud_minus_score_dni
Cloud-vs-DNI component delta.
sun_score_minus_score_cloud
Final score delta vs cloud component.
broken_cloud_flag
Broken-cloud regime indicator.

Trust and Validation

API v1 signals are computed from production model logic and evaluated in SURFRAD-style experiments. Validation focused on beam/occlusion behavior, residual add-value, and transfer across station and time holdouts.

The endpoint exposes intermediate production signals rather than a black-box forecast output. Downstream systems can combine these signals with existing models to apply targeted residual correction and regime-aware decision logic.

Validation Snapshot

  • Evaluated on SURFRAD-style ground irradiance datasets using production model signal outputs.
  • Tested with station-holdout and time-holdout protocols to assess transfer behavior.
  • Beam-state and disagreement signals consistently identify forecast-fragile intervals.
  • Residual-correction usefulness is strongest in transition and broken-cloud regimes.
